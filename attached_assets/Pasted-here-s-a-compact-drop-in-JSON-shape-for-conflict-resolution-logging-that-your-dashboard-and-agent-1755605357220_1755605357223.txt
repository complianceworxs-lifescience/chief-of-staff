here’s a compact, drop-in JSON shape for conflict + resolution logging that your dashboard and agents can both use. It’s terse, consistent, and built for the “one-line summary + one-click intercede” flow.

JSON Schema (concise)
{
  "$id": "cw.conflictEvent.v1",
  "type": "object",
  "required": ["id", "directive_id", "detected_at", "status", "agents", "category", "summary", "decision"],
  "properties": {
    "id": { "type": "string" },                       // uuid
    "directive_id": { "type": "string" },             // links to card
    "detected_at": { "type": "string", "format": "date-time" },
    "resolved_at": { "type": ["string", "null"], "format": "date-time" },

    "status": { "enum": ["auto_resolved", "needs_intervention", "intervened", "dismissed"] },

    "agents": {
      "type": "object",
      "required": ["primary", "counterparty"],
      "properties": {
        "primary": { "type": "string" },              // e.g., "CRO"
        "counterparty": { "type": "string" },         // e.g., "CMO"
        "others": { "type": "array", "items": { "type": "string" } }
      }
    },

    "category": {                                     // what clashed
      "enum": ["revenue", "compliance", "strategy", "timeline", "resourcing", "brand", "data_quality", "other"]
    },

    "trigger": { "type": "string" },                  // short cause phrase
    "metrics": {
      "type": "object",
      "properties": {
        "impact_score": { "type": "number" },         // 0-100
        "effort_score": { "type": "number" },         // 0-100
        "revenue_risk_usd": { "type": "number" },
        "compliance_risk_level": { "enum": ["none","low","medium","high","critical"] }
      }
    },

    "governance_rule": {                              // which rule decided outcome
      "type": "object",
      "properties": {
        "id": { "enum": ["CRO_over_brand","CCO_over_speed","CEO_over_scope","COO_over_method","tie_breaker"] },
        "reason": { "type": "string" }                // brief “why”
      }
    },

    "decision": {
      "type": "object",
      "required": ["winner", "action"],
      "properties": {
        "winner": { "type": "string" },               // e.g., "CRO"
        "action": { "type": "string" },               // e.g., "Run 1-week sales sprint"
        "next_steps": { "type": "array", "items": { "type": "string" } }
      }
    },

    "summary": { "type": "string", "maxLength": 220 },// 1–2 lines for the card
    "notes": { "type": "string" },                    // optional longer text

    "intercede": {                                    // lights your single button
      "type": "object",
      "properties": {
        "enabled": { "type": "boolean" },
        "reason": { "type": "string" },
        "action_id": { "type": "string" }             // opaque id your UI posts back to
      }
    }
  }
}

What the dashboard should render (at a glance)

Banner (only if status ≠ auto_resolved)

⚠ Conflict Detected: {agents.primary} vs {agents.counterparty} — {category}

One-line log inside the directive card

Use summary

Tiny “why” hover

From governance_rule.reason

One button

If intercede.enabled === true, show [Intercede] and POST { action_id, decision: "approve_winner" | "override_to_<AGENT>" | "schedule_followup" , note }

Logbook tab (optional)

Render decision.next_steps as checklist; mark done by agents

Agent behavior (deterministic + simple)

On conflict detection, create event with status: "auto_resolved" if a rule applies AND compliance_risk_level !== "high"; else needs_intervention.

Always fill summary in ≤ 2 sentences.

If needs_intervention, set intercede.enabled = true and provide action_id.

When you click Intercede, agents:

pause directive,

apply your decision,

set status: "intervened" and resolved_at now,

append a new event if outcome triggers follow-up conflicts.

{

This gives you:

a single, consistent JSON your agents emit,

a one-line summary per card,

a single Intercede button wired by intercede.action_id,

clean auditability when you need it.
